  • Creating a Map using a Python Tiling Library hosted in Azure

    The purpose of this task was to create a map of a manufacturing facility in order to effectivly communicate with other departments where data screens where going to be placed in the expansion of the facility. To do so, an AutoCAD drawing was created by the engineering department of the company. In order to get that image to be able to be displayed by Leaflet, the image first needed to go through a process known as 'Tiling'.

    This process effectivly slices the image in many many tiny little pieces that a Library such as Leaflet can stitch together to form a map. To accomplish this, Leaflet has variace zoom levels that the map utilizes, each level having expontentially more images than the previous. These zoom levels are specified when the image is being tiled, in the case of this project, gdal2tiles was used. Once the map and its accompanying webpage was made, the marker system that was denoting where the screens would be placed needed to be tweaked. Before, the map markers were hard-coded for ease of testing. Once everything was verified to be working, the marker data was moved to a MS SQL Server Database that was created using Entity Framework. This allowed anyone in charge of updating the map to edit the marker data with just a datebase update, rather than changing code.

    Now, the expontential aspect of the tiling process ended leading to an interesting dilemma. Initally, the images were being stored alongside the JavaScript code, however, having 10's of thousands of images stored alongside code is not ideal. So a solution had to be found so that the code could still reach the images to properly display the map. To solve this, Microsoft's Azure Blob Storage was utilzed. This Azure service let us store the mass amount of images inside a blob storage that simply could be called from the code to fetch the needed images.

  • Creating an API to handle data requests for a Webpage

    As an onboarding project to get familiar with how all the companies' system worked, a front end application needed to be created as well as an API to go alongside it. The main goal of this project was to create a page that pulled customer data from the database and display various details about the customer and what was ordered. There needed to be a starting page where about 30 of the most recent orders where displayed with the Date, Order ID, Customer ID, etc. were displayed. When you clicked on a table row there were two options that could have been implemented: The first was a redirect to another page that had more details about the customer and the order, the second was to have a popup known as a modal appear and display the data that way.

    As this was the first I project worked on, there was a significant learning curve involved in order to get everything up and running. There was a template provided for the API part of the project, however, ehe entire webpage needed to be set up using Vue, Pinia, Tailwind, and Microsoft's .NET Framework. The API then had to code and setup to work with the webpage and return data to it from API calls. The only call that this project needed was simply GET requests.
